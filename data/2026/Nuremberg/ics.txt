== Notes ==
* https://mc.scrpt.live
* Philip add upcoming shows with Google Spreadsheet and uses a GitHub Action to run a fetch and convert script https://github.com/cowglow/mc-scrpt-live
** Data is all logged via the spreadsheet: event date, start time, venue, event name, event link
*** But unfortunately oftentimes, event link which is a Facebook or silo page, the link breaks and all the information is lost 
* Available as a calendar.ics download
* Unsure how people can subscribe for updates
* Upcoming shows are added on site deployment
* Wants information is open and accessible (available for everyone), without silo or logins
* how can anyone find out when the next/show event is?
* ICS file is available on the site, but no option to subscribe to it (just one-time download)

** Sometimes events are only on IG, Facebook, Tiktok, which means information are locked behind those platforms only
** If users don't have those silo accounts, then they can't access when the next date of an event they're interested in
* {{t}} suggests using webcal:// to trigger calendar subscription feature
** uses h2vx.com converter to convert microformats (here: HTML calendar posts with microformats markup to ics)

* ICS is the main format for calendars
* Google Calendar needs a specific URL for importing events

* https://h2vx.com/ can convert microformats events to ICS for you
** open source: https://indieweb.org/H2VX 

* The event will need the right microformat for the calendar to work correctly
** Use a microformat validator to confirm, https://php.microformats.io/

=== Birthday Calendar ===
** Normal calendar entries for birthdays wouldn't show how many year old someone is
** {{zegnat}} build a script to create birthday calendar entries
*** Creates 5 seperate entries that show persons age (for 2 years in the past and 2 years in the future)
*** Adds additional recurring entry for the future without the age
** Text list with dates and names, script processes the items and makes an ICS
* https://github.com/Zegnat/go-birthday-calendar runs this script everyday with a cronjob
* An ICS can be updated separately to the site

* Q: Any year view or past events?
* Visual presentation options of data
** month grid / calendar view instead of a list
** data: venue, time

* Google spreadsheet has a data source
** yaml, json are alternative
** Google Apps script, https://developers.google.com/apps-script is used to exract the data
** GitHub workflow runs the script and builds using the data for the site
