== Notes ==
* Asking the question about "what can we do for newbies"?
* From students: "I dont know what content to put on there"
* Many (too many?) resources on the wiki
* Indiewebify.me exists and helps introduce folks to microformats
* Can we do more?
* IndieWeb thrives, but is not growing much
* What audience do we want?
** Non-website owners (who are not that skilled in technology)?
** Are people interested in web technology?
* How to get more folks involved? Are there newbies with "technical" website
* Important to suggest "why" - what is valuable
* "Owning your data"
* Easy to push data to Instagram or Facebook
* Express to people why having a personal site is useful versus using silos
* Digital sovereignty is in the zeitgeist right now
* Pain point is getting started

* First thing would be getting domain, which is simple, but everything that follows is a barrier to many
* what is the next way to start?
** learn html and css - students can learn
** WordPress is easy to start and edit and keep online
* What are the other start points
* Even starting with HTML templates might be too much to start, focus on fundamentals like a single HTML file, then slowly add subsidary CSS instead of bombardment of CMS, site generators, etc
* DNS configuration was a big barrier after getting a domain name even when using an AI agent for someone without background information
* Someone nowadays might go to AI agents or LLMs to create "a website for me"
** This might mean missing all the fundamentals of HTML/CSS/JS and how things work on the Internet
* Using AI to create a personal site might use "non-indie" tools which means you get locked in
* If the original intent is to learn about the IndieWeb, break away from silos, then it is important to actually learn and explore on fundamental ideas, becoming a web administrator of sorts 
* In some schools (in LA), there is provided server space + domain names for students to experiment with own online space
** Export before graduating or else content will be deleted
* Other starting points: Tilde-verse, neocities sites (free domain) gives you a simple starting point
* Using a static GitHub page can be an easy way to get started assuming you have some Git knowledge, and some CLI knowledge - additional baggage
* After getting a website online, now what? That's the next question after intial hurdles
** No audience intially compared to using silos 
* Connection is a problem, some don't have feeds or fediverse syndication
** https://micro.blog offers these connection features
* Neocities in the default view when logging in shows that a website is made of a set of files and allows the user to download the full set of files https://artlung.com/blog/2026/06/27/neocities/ 
* IndieWeb Wiki has room to improve and not that user-friendly to true beginners as there's too much information on the Wiki
** Missing a true beginners educational piece

* https://indieweb.org/Getting_Started
* Also, a "tilde" site *can* be used to create a user account on the indieweb wiki https://indieweb.org/User:Tilde.club_~artlung in conjunction with a github account

* The question comes from Tilman's own interest in helping his students join the website
* It's important to know who your audiance is. Maybe there are people who have interest and some who might not have any interest at all
* It's hard to get people who aren't well-versed in this technology. It might be easier to onboard people who are already showing interest.
* Tilman works with student who discuss why it's important to have thier own website
** It's clear that they need to own/buy their own domain
** Should they use start with HTML, CSS, & JS?
* Ideas from the group:
** Start with simple HTML and then step by step add css.
** Learn about Servers to understand how it works
** Being careful with how you get started. One participant tells their story on how going through Cloudflare forced them to setup their DNS. Which is quite an advanced topic
** Starting with a template might be a good stepping stone to get started
** Owning your own domain seems to be, still, the best entry point. 
** Using AI can also help ease the difficulty
** Hosting you own data on a 3rd party can present the issue that your data could be lost. We need to also instroduce the concept of storage and how to perserve data which is actually just hosted for you instead of by you
* One antipattern that might be a hindereance for new comers who are used to the automatic audiance reach from platforms which provide that.
