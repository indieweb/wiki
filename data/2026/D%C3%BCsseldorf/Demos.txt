== Joschi ==
* {{jkphl}}, https://jkphl.is
* Presenting a project separate from website; idea came from 10 years, picking up again
* Apparat - IWC Create Day
** Apparat = very old machine
* Could be base of future website
* Personal knowledge management system
* Built for personal use.
* Wants to create a network of knowledge.
* Has guiding principles — progressive enhancement, local-only, schemaless, plain formats, atomic multi-file operations
* [[Obsidian]] is a bit too low-tech for personal taste
* Made presentation to sort thoughts
* Could include information about people, concepts, books, interviews, etc.
* Wants network to be expansive; not limited.
* Building five layers —
** 0 - raw data
** 1 - parser and graph
** 2 - semantic search
** 3 - ai dialogue
** 4 - web ui
** (0-2 done already, 3 and 4 left)
* New to the AI component; learning as goes.
* Can do semantic search; demo coming up soon.
* Web interface coming soon; CLI-only right now.
* Demoing markdown file which has relationships with other markdown files
* Has 30-40 different documents in vault
* Can list files in CLI
* Has suggestion capability
* has full-text search capability and semantic search too
* semantic search allows for cross-language search
* allows you to find documents similar to another document

== Matthias ==
* http://matthiasott.com
* arrived at noon
* wrote webmention plugin for craftcms several years ago
* last year at IWC demoed new version of webmention plugin
* shows replies to webmentions that were received via bridgy
* if post is mentioned on bluesky, mastodon, etc, gets notification via bridgy
* replies are displayed on personal website
* webmentions can be hard to implement in a way; lots of edge cases, markup styles
* webmentions would fail a lot
* last month built a tool that lists failed webmentions in the back-end
* gives visibility as to why a given webmention did not work
* good debugging tool
* added validation check before something is added to the queue 
** "is this a valid url?"
** screens for sql injection signs
** failed check = not added to queue
* Some webmentions fail because no at:// support.
** now automatically converts at:// urls to http urls
* wouldn’t have discovered without the debugging tool
