== Notes ==
* It's spooky how much the commercial LLM and systems around them know about you.
* Schepp thinks the AI bubble is about to burst this year
  * which might result in (much) higher prices
  * but at this point you might be hooked on them already
  * so, independence from those matters
* https://en.wikipedia.org/wiki/List_of_large_language_models
  * Open weight means they are free to use, but you won't know how (exactly) they were baked.
  * Number of parameters decides about what hardware you need to run them
* https://ai.google.dev/gemma/docs/core
* You need extra RAM for context and headroom for you OS and other apps∂
* Use a mixture of experts models like Gemma 4 26B A4B
* to reduce memory foodprint, use quantization and quantized models like Gemma's SFP8 or Q4_0.
* use lmstudio.ai (for example) to download, install and run your local models
* install OpenWebUI to get a (Chat) interface to you models served via lmstudio.ai
* to know about more current events, attach and configure web search to OpenWebUI
* Fast memory is essential for fast token generation
* CPU and GPU is important for prompt processing.
* comfy.org for image generation
* alternatively to hosting it own your own machines, you can also rent servers (with Mittwald for example)
