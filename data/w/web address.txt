A '''<dfn>web address</dfn>''' is a more friendly and understandable way to refer to a [[domain name]] and what is preferred both in the [[Web sign-in]] user experience, and in New York Times articles about domain names.

== Articles ==
* 2000-01-03 NYT: [https://www.nytimes.com/2000/01/03/business/talk-about-a-pricey-web-address.html Talk About a Pricey Web Address]
* 2001-11-01 NYT: [https://www.nytimes.com/2001/11/01/technology/basics-web-addresses-sprout-new-suffixes-needed-or-not.html BASICS; Web Addresses Sprout New Suffixes, Needed or Not]
* 2014-09-25 NYT: [https://www.nytimes.com/2014/09/26/nyregion/the-rush-is-on-for-web-addresses-that-end-with-panache.html The Rush Is On for Web Addresses That End With Panache]

== Alternatives ==

'''<dfn>DNS handle</dfn>''' or '''<dfn>Domain handle</dfn>''' is using a domain name as a user identifier, and is how user identifiers work in IndieAuth and ATProto.

* "domain handle" has been recently popularized by [[Bluesky]]
** 2025-02-06 Gravatar blog: [https://blog.gravatar.com/2025/02/06/bluesky-handle/ How to Set Up a Custom Domain Handle on Bluesky]

* "internet handle" has been proposed as a term by the site https://internethandle.org/
