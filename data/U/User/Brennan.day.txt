== My IndieWeb Journey ==
I discovered the IndieWeb in December 2025 after discovering [https://omg.lol omg.lol] and writing about it. I decided to get serious about building my own corner of the independent web in January 2026, purchasing brennan.day and building it from scratch with [[Eleventy]].

Since then, I've:
* Built comprehensive IndieWeb implementations including [[IndieAuth]] comments and [[Micropub]] support
* Written extensively about IndieWeb principles and practices for writers and non-technical audiences
* Created 13+ open-source IndieWeb-themed projects and themes
* Joined multiple [[webring]]s and built an extensive [[blogroll]]
* Participated in communities like [https://tilde.town Tilde.town] and omg.lol

== My Website ==
My primary domain is '''brennan.day''', built with [[Eleventy]] since December 2025.

=== IndieWeb Features ===
My site implements:
* ✅ [[A Domain of One's Own]] - brennan.day is my primary identity
* ✅ [[Microformats|Microformats2]] - Full [[h-card]], [[h-entry]], and [[h-feed]] markup
* ✅ [[Webmention|Webmentions]] - Integrated via [[webmention.io]]
* ✅ [[IndieAuth]] - Comment system with PKCE security, allowing users to sign in with their own domain
* ✅ [[Micropub]] - Endpoint for posting from external clients like [[Quill]]
* ✅ [[RSS]] - Available at /feed.xml
* ✅ [[POSSE]] - Using [[Bridgy]] for syndication to social platforms
* ✅ [[Webmention]]-enabled comments stored in Git repository
* ✅ No-JavaScript fallback for [[accessibility]]

== Values & Principles ==
My work is guided by principles documented at https://brennan.day/values/:
* '''Own your content''' - Build for portability, not platform lock-in
* '''Accessibility first''' - Everyone deserves access
* '''[[privacy|Privacy by default]]''' - No tracking, no surveillance
* '''Open source when possible''' - Share knowledge, improve together
* '''Community over growth''' - Center marginalized voices and mutual aid

=== My IndieWeb Principles ===
I've articulated my own principles for the IndieWeb movement:

* '''Good faith code. Good faith writing.''' - The IndieWeb should reject invasive trackers, annoying advertisements, bloated webpages, and [[accessibility|a11y]]-hostile design. We should write from a place of good faith, believing the Internet can still be a tool for empowerment and creativity.
* '''A Pro-social Attitude''' - The web is meant to be social. Sites should have some social element, even if as simple as basic contact information, enabling dialogue rather than one-way broadcasting.
* '''Be Fun. Be Accessible. Be Small.''' - Sites can have personality and express creativity while remaining [[accessibility]] ([[WCAG]] compliant, screen reader friendly) and performant (compressed images, lazy loading, progressive enhancement, no-JavaScript fallback).

See [https://brennan.day/posts/2026-02-14-what-is-the-indieweb/ What is the IndieWeb?] for my full explanation.
