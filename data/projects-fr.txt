= Projets Indie Web =

Une liste de projets supportés ou activement utilisés par les [[Guest_List|participants]] à l'IndieWebCamp pour créer l'Indie Web.

La page est découpée en production, expérimental, hacks et explorations tout comme d'autres pour des projets qui sont ou semblent en rapport avec l'IndieWeb mais ne sont soit pas en cours d'utilisation par les participants ou dont le statut est inconnu.

Classés par niveau de qualité, les projets sont listés :
* En priorité sur la liste les systèmes d'hébergement de contenus / blogging, les systèmes d'hébergement de contenus supplémentaires (par ex. ceux dédiés uniquement à un type de contenu), et puis les autres projets indieweb utiles de blocs de construction.
* Puis par nombre de participants + autres indépendants utilisant chaque projet.

Trié dans chaque niveau de qualité par nombre de participants IndieWebCamp les utilisant pour leur propre site (par conséquent vous pouvez rapidement voir quels projets sont les plus "réels", en usage et probablement bien supportés).

Voir aussi en dehors du site : 
* [http://libreprojects.net Libre Projects] - logiciel web open source utilisable, hébergé
* [https://github.com/indieweb github.com/indieweb organization] - repositories en rapport avec l'indieweb. Une fois que Mime ajoutera les  [[2012/Guest_List|participants 2012]] à l'organisation, nous pourrons y bifurquer nos projets de façon à ce que nous puissions suivre les progrès de tous !

=== sadlittlewebjournal ===

[http://jere.my/?page=writings&sub_page=archive&display=5 sadlittlewebjournal] est un weblog écrit en Perl qui utilise PostgreSQL ou MySQL et une interface directement en ASCII. La maintenance du site est faite via un backend intuitif qui permet d'ajouter, effacer et modifier les entrées. D'autres fonctionnalités sont un livre d'or intégré, un chart de stats web complet avec des barres de graphe en ASCII et différentes autres fonctionnalités modulaires.

La fonctionnalité actuelle POSSE comprend la publication de nouveaux posts vers un flux Twitter ou StatusNet externe, mais l'alternative PESOS est aussi supporté : republier des posts syndiqués à partir d'un tel flux. Sur le plan de route, les commentaires utilisant webmentions et les micrformats.

* [http://jere.my/files/software/unix/sadlittlewebjournal Prenez le code]

Les enthousiastes Indieweb qui l'utilisent actuellement sur leurs propres sites :

=== Storytlr ===
{{main-fr|Storytlr}}

[http://storytlr.org Storytlr] est une plate-forme open-source de flux-de-vie. Elle syndique automatiquement le contenu provenant de nombreux services sociaux  (twitter, youtube, flickr, etc.) et supporte la publication de contenu avec une syndication vers Twitter. Un outil génial pour les approches [[POSSE-fr|POSSE]] ou [[PESOS-fr|PESOS]] vers l'indieweb. Sur la roadmap : un client mobile (tout spécialement pour poster et syndiquer des images), plus de destinations de syndication, support pour le commentaire fédéré avec Salmon.

=== Federated Community ===

L'idée ici est d'utiliser l'email-basé-sur-pop, un script générique python et du html statique pour paramétrer un site dynamique de microblogging et d'hébergement de photo. Le workflow actuel comprend l'envoi d'un email vers un compte e-mail pop. Puis quand le script phython trouve le nouvel email, il télécharge l'email vers mon serveur html statique et envoie aussi un email à mes comptes facebook et twitter pour syndication. Actullement, le script python est situé [https://github.com/geo-gs/federatedcommunity/tree/python ici] et un exemple de html statique est situé [https://github.com/geo-gs/gcarlson.github.com ici]. Prochaine étape, ajouter la capacité de téléverser des albums photos. Après cela, les commentaires.

Les personnes utilisant actuellement cela sur leurs propres sites : 
* [http://geo.gs George Em]

==== IRC ====
[[File:irc-personal-hub.png|300px|right]]
[[User:Aaronpk|Aaron Parecki]] utilise un serveur privé IRC avec plusieurs canaux comme hub de communication personnel. Ce projet n'a pas de nom spécifique, et n'a pas de base de code unique, et demeure hautement expérimental. Néanmoins il l'utilise et le développe depuis presque trois ans.

Le [http://aaronparecki.com/Loqi bot] dans le canal IRC peut contrôler les lumières dans la maison, faire du text to speech sur les ordinateurs de la maison, affiche les citations Twitter et les modifications wiki, fait des conversions d'unités et d'autres calculs, gère une todo list, et parfois fait des remarques impertinentes.

Sa structure modulaire l'a rendu extrêmement facile pour y ajouter rapidement de nouvelles fonctionnalités, et a probablement ralenti le développement d'Aaron sur des équivalents basés sur le web plus accessibles.

==== IndieWeb Messaging ====
But : pouvoir envoyer à quelqu'un un message court en ne connaissant que son nom de domaine. Il devrait pouvoir recevoir le message comme il le veut (SMS, Email, Twitter DM, etc) sans que l'expéditeur ne sache par quel médium le message sera distribué.
* Détails : http://aaronparecki.com/IndieWeb_Messaging
* Source Code : https://github.com/aaronpk/IndieWeb-Messaging

=== Diaspora ===
{{main-fr|Diaspora-fr}}

[[Diaspora-fr|Diaspora]] est un projet open source pour héberger un réseau social sur votre propre serveur qui [[fédération-fr|fédère]] avec d'autres instance Diaspora, lesquelles sont appelés "pods". 

Aucun participant ne l'utilise à cette heure sur son propre site.

=== Ghost ===
{{main-fr|Ghost-fr}}

Ghost est un "concept idéaliste et fictionnel pour un fork WordPress-lite" [http://john.onolan.org/ghost/] that is [http://www.kickstarter.com/projects/johnonolan/ghost-just-a-blogging-platform/widget/video.html cherchant un financemement sur kickstarter].

Aucun participant ne l'utilise à cette heure sur son propre site.

=== OStatus pour WordPress ===
[http://wordpress.org/extend/plugins/ostatus-for-wordpress/ OStatus for WordPress] est une collection d'extensions pour faire que les blogs  WordPress soient suivables par status.net et d'autres instances OStatus.

Aucun participant ne l'utilise sur son propre site.

Si nous pouvions avoir quelque participation indieweb par quelques types utilisant OStatus, ce serait génial de comprendre comment cela fonctionne à ce jour.


=== Shaarli ===
[http://sebsauvage.net/wiki/doku.php?id=php:shaarli Shaarli] est une installation clone minimaliste de [http://delicious.com delicious] que vous pouvez installer sur votre propre site web. Elle est conçue pour être personnelle (utilisateur-unique), rapide et pratique.

Elle est prioritairement une application de bookmarking, mais la liste de fonctionnalités indique que vous pouvez aussi l'utiliser pour du " .... micro-blogging (like Twitter), a pastebin, an online notepad, a snippet repository, etc."
 	
Aucun participant dne l'utilise sur son propre site.
* [[User:Gregorlove.com|gRegor Morrill]] est intéressé pour l'essayer et tester son usage pour le micro-blogging.

== abandonnés ==
Les projets qui peuvent avoir supporté des sites indie web dans le passé mais qui semble avoir été abandonnés :

=== glow ===
[http://blog.glow.io glow] (dernier post 2011-07-12 conseille d'utiliser G+) supporte les intégrations avec Twitter et Facebook et une pile complète de protocoles du web social fédéré et de standards (activitystrea.ms, poco, pubsubhubub, salmon, etc). 

Glow est acutellement limité aux sous-domaines glow.io si vous voulez l'essayez. À un certain stade, il sera probablement ouvert à n'importe quel hébergement ou domaine.

Les types qui l'ont utilisé sur leur propre site : 
* [[User:DBounds|Darren Bounds]] (http://dbounds.glow.io/profile (connexion time out), auteur de glow)
* ...

Autres instance / travaux antérieurs : 
* <span id="ooava"><nowiki>http://ooava.com/</nowiki></span> (domaine abandonné) était un projet personnel de noeud social pour publier (photos, videos, status updates) et converser sur le web et Android. ooava était un site personnel qui a évolué dans le projet glow.
