La capacité de publier et s'abonner à des flux de contenus est un bloc de construction vital de l'indieweb.

== Travaux existants ==

* [[ActivityStreams-fr|ActivityStreams]] - un effort pour créer un schéma pour les flux sociaux basé sur JSON et ATOM
* [http://atomenabled.org/ ATOM] - un schéma XML pour publier des flux
* [https://code.google.com/p/pubsubhubbub/ PuSH] - un protocole pour publier et s'abonner à des flux pour les flux en temps réel

== Comment-faire ==

* [[How_to_set_up_your_realtime_feed-fr|Comment paramétrer votre flux temps réel]] - beaucoup d'information utile en rapport avec l'installation et le réglage d'un flux ATOM+ActicityStreams+PuSH
* [[syndication formats-fr#Formater le Contenu pour la Syndication|Comment formater le contenu pour les flux]]

== Futur ==
=== Les Flux sont-ils en train de mourir  ===
Les APIs de Feedburner ferment le 2012-10-20 (selon https://developers.google.com/feedburner/ ). Voir aussi : http://pleasedontkillfeedburner.com/

Est-il temps de déprécier les flux, ou au moins de laisser tomber toute dépendance sociale fédérée sur les flux ?

Avec [http://microformats.org/w/uf2-fr#h-entry microformats2 h-entry], les [http://microformats.org/w/uf2-fr#implementations parseurs microformats2] pour Node et PHP y-a-t-il besoin encore de flux/formats <s>XML</s>JSON quand vous pouvez recevoir toute page HTML sous forme de JSON ?
