=== Header Behaviour ===
* Anchor IDs on subheadings, usually a symbol like a emoji link icon, or a symbol e.g.
** https://en.wikipedia.org/wiki/Pilcrow or https://en.wikipedia.org/wiki/Section_sign
* https://www.w3docs.com/snippets/html/how-to-create-an-anchor-link-to-jump-to-a-specific-part-of-a-page.html
* https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/a
* There's a "Selected Text"/ "Psudo anchor links" method if the heading doesn't use `id=heading-identifier`, which is supported in Safari/Chrome/Firefox
** Ross's retro site URL above is an example of this type of `#:~:text=` link
** https://developer.mozilla.org/en-US/docs/Web/URI/Reference/Fragment/Text_fragments
** The extension Ross imagined but has not installed.  Get the ID of an element in the context menu: https://addons.mozilla.org/en-US/firefox/addon/get-element-id/
** Naty starting using anchor links on her site because of the table of content feature

== New CSS for light/dark modes ==
* Relatively new CSS function `light-dark()`
https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/Values/color_value/light-dark
** https://css-tricks.com/almanac/functions/l/light-dark/
** https://rolandfranke.nl/frontend-stories/css-light-dark-color-function/
* Mike uses @media queries and reads the user's "prefers light or dark mode"
* Setting variables and symbolic name like var(--header-color) is good for when the color needs to be referenced multiple times
* `light-dark()` would be useful to reduce code if it's just declared once
