''Cette page a démarré sur [[principles]]''

La Communauté IndieWeb se fonde essentiellement sur des  '''<dfn>principes</dfn>''' (AKA <dfn>tenets (doctrine ?)</dfn>) tels que [[own your data-fr|possède tes données]], gratte toi-même ce qui démange, construis des outils pour toi-même, [[selfdogfood-fr|selfdogfood]], documente tes trucs, open-source tes trucs, le [[design-fr|design]] d’[[UX-fr|UX]] est plus important que les protocoles, data visible en priorité pour les humains et les machines passent après, agnostique aux plateformes, [[plurality-fr|pluralité]] sur monoculture, [[longevity-fr|longévité]] et se souvenir de nous [[fun-fr|amuser]] !

L’IndieWebCamp a un [[code-of-conduct-fr|code de conduite]].

== Principes-clés ==
<span style="float:right;height:192px;font-size:192px;margin:96px 0 -32px">🗽</span>
‘’’Les principes-clés de construction sur l’indie web''', numérotés pour référence, pas nécessairement par ordre de priorité.

# ✊ '''[[own-your-data-fr|Own your data]]'''.
# 🔍 Utilisez de la '''data visible''' d'abord pour les humains, les machines en second. Voir aussi [[DRY-fr|DRY]]
# 💪 Construisez les '''outils pour vous-même''', pas pour tous vos amis. Il est extrêmement difficile de combattre [http://fr.wikipedia.org/wiki/Loi_de_Metcalfe la loi de Metcalfe] : vous ne pourrez pas convaincre tous vos amis de rejoindre le web indépendant. Mais si vous construisez quelque chose qui satisfait vos propres besoins, mais qui soit aussi rétro-compatible pour les personnes qui ne l'ont pas rejoint (disons en pratiquant [[POSSE-fr|POSSE]]), le temps et l'effort que vous avez mis à construire vos propres outils n'est pas perdu pour la simple raison que les autres ne l'ont pas encore rejoint à cette heure.
# 😋 Mangez votre propre '''[[dogfood-fr|nourriture pour chiens]]'''. Quel que soit le truc que vous construisez, ce devrait être pour vous-même. Si vous n'en dépendez pas, pourquoi quelqu'un d'autre le ferait ? Nous appelons ça le [[selfdogfood-fr|selfdogfood]]ing. Plus important, construisez l'indie web autour de vos besoins. Si vous concevez des outils pour n'importe quel utilisateur hypothétique, ces outils ne peuvent pas véritablement exister. Le [[selfdogfooding]] est aussi une forme de "[https://fr.wikipedia.org/wiki/Preuve_de_travail preuve de travail] » pour aider à se concentrer sur des interactions productives.
# 📓 '''Documentez''' vos trucs. Vous avez construit un endroit pour faire parler votre esprit, utilisez-le pour documenter vos processus, idées, designs et code. Documentez-le au moins pour votre futur vous.
# 💞  '''Open sourcez''' vos trucs !  Vous n'êtes bien sûr pas obligé, mais si vous aimez l'existence de l'indie web, placer votre code en open source signifie que d'autres personnes peuvent rejoindre l'indie web plus rapidement et plus facilement.
# 📐 '''L'[[UX-fr|UX]] et le [[design-fr|design]] sont plus importants''' que les protocoles. Nous nous concentrons en priorité sur l'UX, et puis nous comprenons de cela comment nous construisons/développons/découpons les protocoles minimums les plus simples et suffisants pour supporter cette UX, et rien de plus. Aussi formulé sous UX avant la plomberie.
# 🌐 Construisez des '''plates-formes agnostiques aux plates-formes.''' Plus votre code sera modulaire et composé de pièces que vous pourrez échanger, moins vous serez dépendant d'un terminal spécifique, d'une UI, d'un langage de modélisation, d'un langage backend, d’un modèle de stockage, de database, de plate-forme. Plus votre code est modulaire, plus la chance sera élevée qu’il y aura au moins un peu de ce code qui pourra être ré-utilisé, amélioré et que vous pourrez ensuite réincorporer.
# 🗿 '''[[Longevity-fr|Longévité]]'''. Construisez pour le  '''[http://adactio.com/articles/1522/ long web]'''. Si la société humaine sait préserver les anciens papyrus, les photographies victoriennes et les os de dinosaures, nous devrions pouvoir construire une technologie web qui ne nous oblige pas au nom du progrès, à démolir tout ce que nous avons produit durant quelques années.
# ✨ <span id="Pluralité">'''[[plurality|Pluralité]]'''</span>. Avec IndieWebCamp nous avons spécifiquement choisi d'encourager et d'embrasser une diversité d'approches et d’implémentations. Ce <span id=« Contexte »>contexte</span> rend l’IndieWeb plus fort et plus résilient que toute autre approche (souvent [[monoculture-fr|monoculture]]).
# 🎉 '''Poilons-nous'''<ref>Have '''Fun'''!</ref>. Vous vous souvenez de la page GeoCities que vous avez construite mi-90 ? Celle avec des applets Java, des couleurs d'arrière-plan vert fluo et plusieurs GIFs animés ? Ce peut avoir été laid, mal codé et craignosse, mais c'était "fun, bordel". Gardons le web étrange et intéressant.
