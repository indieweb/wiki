'''<dfn>pixel art</dfn>''' are images that are intentionally low-resolution, sometimes used as icons on indieweb sites. These pixel art images should be scaled up using nearest neighbor scaling.

== How ==
=== How to create ===
* Jenn Schiffer's pixel art site https://make8bitart.com/

=== How to render ===
Use the <code>image-rendering: pixelated</code> CSS property so that pixel art scales as intended. [https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/Properties/image-rendering MDN documentation]

<pre>
.pixelated {
    image-rendering: pixelated;  /* Chrome */

    /* Firefox */
    image-rendering: crisp-edges;
    image-rendering: -moz-crisp-edges;

    image-rendering: -webkit-optimize-contrast;  /* Safari */
}
</pre>
* Note: this example CSS selector was previously <code>.u-pixel-art</code> which may be useful for a [[microformats]] parser, although we are not currently aware of a consuming use-case for it currently.

See lospec's guide ([https://web.archive.org/web/20190406044603/https://lospec.com/browser-image-scaling-comparison archived]) on all the pixel art scaling options.
